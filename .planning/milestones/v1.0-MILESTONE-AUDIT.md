## Integration Check

**Cross-Phase Wiring: 22/22 connections verified**

All API routes have consumers. All data contracts match between producers and consumers. Session cookie flows correctly through the full OAuth -> fetch -> submit -> persist chain. Athlete JSON schema written by submit-result.js exactly matches what athlete-loader.ts reads.

| Route | Function | Consumer | Status |
|-------|----------|----------|--------|
| /api/strava-auth | strava-auth.js | submit.astro (401 redirect), error.astro | WIRED |
| /api/strava-callback | strava-callback.js | Strava platform (redirect_uri) | WIRED |
| /api/strava-fetch-activity | strava-fetch-activity.js | submit.astro fetch POST | WIRED |
| /api/submit-result | submit-result.js | submit-confirm.astro fetch POST | WIRED |
| /api/strava-webhook | strava-webhook.js | Strava platform (deauth events) | WIRED |
| /api/health | health.js | Ops (no frontend caller — intentional) | WIRED |

### Data Contract Consistency

| Field | Written (submit-result.js) | Read (athlete-loader.ts) | Match |
|-------|---------------------------|--------------------------|-------|
| athleteId | body.athleteId (string) | athlete.athleteId -> rider.id | YES |
| displayName | body.name.trim() | athlete.displayName -> rider.name | YES |
| category | body.category | filtered by categoryIds | YES |
| day1.movingTimeSeconds | Number(body.movingTimeSeconds) | athlete.day1!.movingTimeSeconds | YES |
| day2.sectorEfforts | body.sectorEfforts || {} | SECTOR_SEGMENT_IDS.map over day2.sectorEfforts | YES |
| day2.komEfforts | body.komEfforts || {} | KOM_SEGMENT_IDS.map over komEfforts | YES |
| day2.komSegmentIds | body.komSegmentIds || [] | fallback path when komEfforts absent | YES |

### UX Gap (Non-Critical)

**Submission success acknowledgment:** submit-confirm.astro redirects to `/?submitted=true` after a successful submission, but index.astro does not read this query parameter. The rider lands on the home page with no visual confirmation. Data integrity is unaffected — the JSON was written and build hook fired. This is a polish item, not a blocker.
