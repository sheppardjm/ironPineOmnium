# Milestone v1.0: MVP

**Status:** SHIPPED 2026-04-08
**Phases:** 1-12
**Total Plans:** 33

## Overview

The existing static Astro site had a complete scoring engine and sample-data leaderboard. This milestone added the three capabilities that made it real: Strava OAuth authentication, a submission pipeline that fetches and validates activity data, and persistent storage of rider results that drives the public leaderboard. Two blockers — Strava API compliance and the athlete limit review — were resolved first. The build order was gated strictly: compliance first, then infrastructure, then OAuth, then submission, then persistence, then display enhancements.

## Milestone Summary

**Decimal Phases:** None — all phases were integer-numbered.

**Key Decisions:**

| Decision | Rationale | Outcome |
|----------|-----------|---------|
| Single submission site | Avoids cross-site data sharing and duplicate OAuth flows | Correct — simplified architecture |
| Trust-based submissions | Small community event, no need for admin overhead | Correct — no issues |
| Strava athlete ID as rider identity | Natural link between day 1 and day 2 activities | Correct — clean association |
| GitHub Contents API for persistence | Static site can read JSON at build time, no database needed | Correct — simple and effective |
| Netlify Functions v1 ESM | Compatibility with existing Netlify setup | Correct — all functions work |
| Deferred Strava review to Phase 12 | Needed finished UI for screenshots | Correct — submitted with real screenshots |
| Editorial race-poster redesign | Previous dark theme felt generic | Correct — distinctive visual identity |

**Issues Resolved:**
- Route map images swapped between Day 1 and Day 2 events (fixed in Phase 11)
- Dead code accumulation: sample-data.ts, LogoMark.astro, dead hidden fields (cleaned in Phase 11)
- Strava athlete name not threaded to confirm page (fixed in Phase 5 gap closure)
- ARM64 esbuild binary mismatch during netlify dev (resolved with pnpm override in Phase 2)

**Issues Deferred:**
- DSGN-02: Companion site links (requires changes in separate repos: mkUltraGravel, hiawathasRevenge)
- submitted=true query param not consumed by index.astro (no success banner after submission)
- Dead exports in segments.ts (ALL_SCORED_SEGMENT_IDS, SEGMENT_LABELS, type unions)

**Technical Debt Incurred:**
- Phase 10 has no VERIFICATION.md or SUMMARY.md files (work done via frontend-design skill)
- scoreCategory() produces Infinity benchmarks with empty riders (UI-guarded)
- loadAthleteResults() called 3x at build time (harmless for static)
- strava_api_error gets generic fallback message in submit.astro
