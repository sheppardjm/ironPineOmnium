**HTML structure (in the Astro template):**

- A `<form id="submit-form">` with:

  - `<button type="submit" id="submit-btn" class="primary-button">Look up activity</button>`
- A `<p id="submit-error" class="submit-error" hidden></p>` for inline error display
- A `<p id="submit-loading" class="submit-loading" hidden>Fetching activity from Strava...</p>` for loading state

**Client-side script (in a `<script>` tag, NOT `is:inline`):**

The script handles form submission:

1. On form submit (preventDefault):
   - Hide any previous error, show loading indicator, disable button
   - Read the input value
   - POST to `/api/strava-fetch-activity` with `{ activityUrl }` as JSON body, include `credentials: "same-origin"` so cookies are sent
   - If response status is 401: redirect to `/api/strava-auth` (rider needs to authenticate first)
   - Parse JSON response
   - If response has `error` field: map error codes to user-friendly messages and show in `#submit-error`:
     - `invalid_url`: "That doesn't look like a Strava activity URL. It should look like https://www.strava.com/activities/12345678"
     - `activity_not_found`: "We couldn't find that activity. Make sure it exists and isn't set to private."
     - `wrong_athlete`: "That activity belongs to a different Strava account. You can only submit your own activities."
     - `wrong_date`: "That activity is from {actualDate}. The Iron & Pine Omnium is June 6-7, 2026."  (read `actualDate` from response JSON)
     - `rate_limited`: "Strava's API rate limit has been reached. Please wait a minute and try again."
     - `network_error`: "Network error reaching Strava. Check your connection and try again."
     - default: "Something went wrong. Please try again."
   - Re-enable button and hide loading after error
   - If response has NO error (success): base64url-encode the response JSON and redirect:

2. base64url encoding function (inline in the script):
```javascript
function toBase64url(obj) {
  const json = JSON.stringify(obj);
  return btoa(json).replace(/\+/g, "-").replace(/\//g, "_").replace(/=/g, "");
}
```
(The payload is all ASCII — activityId, athleteId are numeric strings, dates are ISO, sectorEfforts are number values — so plain btoa is safe without TextEncoder.)

3. Redirect: `window.location.href = "/submit-confirm?payload=" + toBase64url(data);`
