<task type="checkpoint:human-action" gate="blocking">
  <name>Task 2: Register Strava webhook subscription</name>
  <action>
The Strava webhook subscription must be registered via a one-time curl command from the developer's machine (or the Strava API Explorer). This requires the production callback URL and the Strava app credentials.

**Prerequisites:**
- The site must be deployed to Netlify so the webhook URL is publicly accessible.
- The STRAVA_VERIFY_TOKEN env var must match what's configured in Netlify.

**Registration command:**
```bash
curl -X POST https://www.strava.com/api/v3/push_subscriptions \
  -F client_id=${STRAVA_CLIENT_ID} \
  -F client_secret=${STRAVA_CLIENT_SECRET} \
  -F callback_url=https://YOUR-SITE.netlify.app/api/strava-webhook \
  -F verify_token=${STRAVA_VERIFY_TOKEN}
```

Replace `YOUR-SITE` with the actual Netlify site hostname.

**Expected response:** `{ "id": 12345 }`

**After registration:**
- Note the subscription ID in the plan summary for future reference.
- The webhook is now active — deauth events from Strava will be sent to the callback URL.

**To verify the handshake worked:**
```bash
curl -s "https://www.strava.com/api/v3/push_subscriptions?client_id=${STRAVA_CLIENT_ID}&client_secret=${STRAVA_CLIENT_SECRET}" | jq .
```
This should return the registered subscription.
  </action>
