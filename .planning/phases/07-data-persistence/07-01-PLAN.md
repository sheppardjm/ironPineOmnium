  <name>Task 1: Create netlify/functions/submit-result.js</name>
  <files>netlify/functions/submit-result.js</files>
  <action>
Create a new Netlify Functions v1 ESM handler (`export const handler`) that:

**Imports:** `import { parse } from "cookie-es";` (no serialize needed — this function only reads cookies, no Set-Cookie).

**Session validation:**
1. Read `strava_session` cookie using `parse(event.headers.cookie || "")`.
2. JSON.parse the cookie value. If missing or unparseable, return `{ statusCode: 401, body: JSON.stringify({ error: "no_session" }) }`.
3. Extract `session.athleteId`.

**Request parsing:**
4. Only accept POST. If `event.httpMethod !== "POST"`, return 405.
5. Parse `event.body` as JSON. Extract: `name`, `category`, `activityId`, `athleteId`, `movingTimeSeconds`, `startDateLocal`, `sectorEfforts`, `komSegmentIds`, `komEfforts`.
6. Basic validation: `name` must be non-empty string (trimmed), `category` must be one of `"men"`, `"women"`, `"non-binary"`, `activityId` must be non-empty string, `athleteId` must be non-empty string.
7. If validation fails, return `{ statusCode: 200, body: JSON.stringify({ error: "invalid_payload" }) }`.

**Athlete ID mismatch check (SECURITY):**
8. Compare `body.athleteId` against `session.athleteId`. If they differ, return `{ statusCode: 403, body: JSON.stringify({ error: "athlete_mismatch" }) }`.

**GitHub Contents API — GET existing file:**
9. Define constants:
   ```js
   const GITHUB_API = "https://api.github.com";
   const owner = process.env.GITHUB_OWNER;
   const repo = process.env.GITHUB_REPO;
   const filePath = `public/data/results/athletes/${body.athleteId}.json`;
   const ghHeaders = {
     Authorization: `Bearer ${process.env.GITHUB_TOKEN}`,
     Accept: "application/vnd.github+json",
     "X-GitHub-Api-Version": "2022-11-28",
     "Content-Type": "application/json",
   };
   ```
10. GET `${GITHUB_API}/repos/${owner}/${repo}/contents/${filePath}` with ghHeaders.
11. If GET returns 200, parse the response JSON, extract `.sha` and decode `.content` (base64) via `Buffer.from(existing.content, "base64").toString("utf-8")` then JSON.parse to get `existingData`.
12. If GET returns 404, `existingData = null` and `sha = undefined`.
13. If GET returns any other status, return `{ statusCode: 200, body: JSON.stringify({ error: "github_error" }) }`.

**Deduplication check:**
14. If `existingData` is not null and `activityId !== "manual"`:
    - If `existingData.day1?.activityId === body.activityId` OR `existingData.day2?.activityId === body.activityId`, return `{ statusCode: 200, body: JSON.stringify({ ok: true, duplicate: true }) }` — no write needed.

**Build the athlete JSON object:**
15. Determine which day based on `body.startDateLocal`:
    - `"2026-06-06"` = day1
    - `"2026-06-07"` = day2
    - Any other date: return `{ statusCode: 200, body: JSON.stringify({ error: "invalid_date" }) }`.

16. `const now = new Date().toISOString();`

17. If `existingData` is null (new athlete file):
    ```js
    const athleteData = {
      athleteId: body.athleteId,
      displayName: body.name.trim(),
      category: body.category,
      day1: null,
      day2: null,
      createdAt: now,
      updatedAt: now,
    };
    ```
    Then set the appropriate day object (see step 18).

18. If `existingData` is not null (updating existing file):
    - **Identity lock:** Use `existingData.displayName` and `existingData.category` — ignore `body.name` and `body.category`.
    - Merge:
    ```js
    const athleteData = {
      ...existingData,
      updatedAt: now,
    };
    ```
    Then set the appropriate day object.

19. Day object construction:
    - For day1: `athleteData.day1 = { movingTimeSeconds: Number(body.movingTimeSeconds), activityId: body.activityId, submittedAt: now };`
    - For day2: `athleteData.day2 = { sectorEfforts: body.sectorEfforts || {}, komSegmentIds: body.komSegmentIds || [], komEfforts: body.komEfforts || {}, activityId: body.activityId, submittedAt: now };`

**GitHub Contents API — PUT:**
20. Construct the PUT body:
    ```js
    const putBody = {
      message: `submission: ${body.athleteId} ${isDay1 ? "day1" : "day2"}`,
      content: Buffer.from(JSON.stringify(athleteData, null, 2)).toString("base64"),
      ...(sha ? { sha } : {}),
    };
    ```
21. PUT to `${GITHUB_API}/repos/${owner}/${repo}/contents/${filePath}` with ghHeaders and JSON.stringify(putBody).
22. If PUT returns 409 (SHA conflict), retry once: re-GET to fetch new SHA, then re-PUT. If retry also fails, return `{ statusCode: 200, body: JSON.stringify({ error: "write_conflict" }) }`.
23. If PUT returns any non-ok status (after retry logic), return `{ statusCode: 200, body: JSON.stringify({ error: "github_write_error" }) }`.

**Trigger build hook (fire-and-forget):**
24. After successful PUT:
    ```js
    const hookUrl = process.env.NETLIFY_BUILD_HOOK;
    if (hookUrl) {
      fetch(hookUrl, { method: "POST", body: "{}" }).catch(() => {});
    }
    ```
    Do NOT await the result. The `.catch(() => {})` silences unhandled promise rejection.

**Return success:**
25. Return `{ statusCode: 200, headers: { "Content-Type": "application/json" }, body: JSON.stringify({ ok: true }) }`.

**Important patterns to follow:**
- Use Netlify Functions v1 ESM syntax: `export const handler = async (event, _context) => { ... };`
- Add the standard file header comment matching other functions (see strava-fetch-activity.js).
- All user-facing errors use HTTP 200 with `{ error: "code" }` pattern (consistent with strava-fetch-activity.js). Only session/auth failures use non-200.
- Do NOT import or call getValidAccessToken — submit-result does not make Strava API calls, it only reads the session cookie for the athleteId.

In `src/pages/submit-confirm.astro`, replace the placeholder submit handler in the `setupForm()` function. Currently, the form submit handler at the comment "Phase 7 will POST to /api/submit-result" shows a placeholder message "Submission ready" and disables the button.

Replace that placeholder section (everything after the `if (!category)` validation block, starting from the comment `// Phase 7 will POST to /api/submit-result.`) with a real fetch POST:

```typescript
// Disable button and show submitting state
const btn = document.getElementById("confirm-btn") as HTMLButtonElement;
btn.textContent = "Submitting...";
btn.disabled = true;

// Collect form data from hidden inputs
const formEl = document.getElementById("confirm-form") as HTMLFormElement;
const fd = new FormData(formEl);

const payload = {
  name: name,
  category: category,
  activityId: fd.get("activityId"),
  athleteId: fd.get("athleteId"),
  movingTimeSeconds: Number(fd.get("movingTimeSeconds")),
  startDateLocal: fd.get("startDateLocal"),
  sectorEfforts: JSON.parse((fd.get("sectorEfforts") as string) || "{}"),
  komSegmentIds: JSON.parse((fd.get("komSegmentIds") as string) || "[]"),
  komEfforts: JSON.parse((fd.get("komEfforts") as string) || "{}"),
};

try {
  const res = await fetch("/api/submit-result", {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify(payload),
    credentials: "include",
  });

  const result = await res.json();

  if (result.ok) {
    // Redirect to homepage with submitted flag
    window.location.href = "/?submitted=true";
    return;
  }

  // Show error and re-enable button
  btn.textContent = "Confirm Submission";
  btn.disabled = false;

  const errorMap: Record<string, string> = {
    athlete_mismatch: "Session mismatch — please sign in again.",
    invalid_payload: "Missing required fields. Please try again.",
    invalid_date: "Activity date is outside the event window.",
    write_conflict: "Temporary conflict — please try again.",
    github_error: "Server error — please try again in a moment.",
    github_write_error: "Failed to save — please try again.",
  };

  const msg = errorMap[result.error] || "Something went wrong. Please try again.";
  alert(msg);

} catch (_err) {
  btn.textContent = "Confirm Submission";
  btn.disabled = false;
  alert("Network error — please check your connection and try again.");
}
```

Do NOT change any other part of the file — leave renderPreview, populateHiddenFields, fromBase64url, formatTime, and the DOMContentLoaded handler exactly as they are. Leave all styles untouched.
