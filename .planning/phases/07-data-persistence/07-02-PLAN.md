Create a new Netlify Functions v1 ESM handler (`export const handler`).

**No imports needed** — this function uses only fetch() and Buffer (both built-in). No cookie parsing, no strava-tokens.

**Constants:**
```js
const GITHUB_API = "https://api.github.com";
```

**Handler structure — dual GET/POST:**

The handler inspects `event.httpMethod` and routes accordingly.

**GET handler (subscription handshake):**
1. Read `event.queryStringParameters`.
2. If `hub.mode === "subscribe"` AND `hub.verify_token === process.env.STRAVA_VERIFY_TOKEN`:
   - Return `{ statusCode: 200, headers: { "Content-Type": "application/json" }, body: JSON.stringify({ "hub.challenge": params["hub.challenge"] }) }`.
3. Otherwise return `{ statusCode: 403, body: "Forbidden" }`.

**POST handler (webhook events):**
1. Parse `event.body` as JSON. If unparseable, return `{ statusCode: 200, body: "EVENT_RECEIVED" }` (don't reject — Strava will retry).
2. Detect deauth event: `payload.object_type === "athlete"` AND `payload.aspect_type === "update"` AND `(payload.updates?.authorized === "false" || payload.updates?.authorized === false)`. Check BOTH string "false" and boolean false — Strava sends string but check both defensively.
3. If deauth detected:
   - Extract `athleteId = String(payload.owner_id)`.
   - Construct `filePath = public/data/results/athletes/${athleteId}.json`.
   - **GET the file to retrieve SHA:**
     ```js
     const owner = process.env.GITHUB_OWNER;
     const repo = process.env.GITHUB_REPO;
     const ghHeaders = {
       Authorization: `Bearer ${process.env.GITHUB_TOKEN}`,
       Accept: "application/vnd.github+json",
       "X-GitHub-Api-Version": "2022-11-28",
       "Content-Type": "application/json",
     };
     ```
     GET `${GITHUB_API}/repos/${owner}/${repo}/contents/${filePath}`.
   - If GET returns 404, the file is already gone — no action needed. Skip to return 200.
   - If GET returns 200, extract `.sha` from the response.
   - **DELETE the file:**
     DELETE `${GITHUB_API}/repos/${owner}/${repo}/contents/${filePath}` with body `JSON.stringify({ message: "deauth: remove athlete ${athleteId}", sha })`.
   - If DELETE fails (non-200, non-404), log to console.error but do NOT return an error to Strava — always return 200.
   - **Trigger build hook (fire-and-forget):**
     ```js
     const hookUrl = process.env.NETLIFY_BUILD_HOOK;
     if (hookUrl) {
       fetch(hookUrl, { method: "POST", body: "{}" }).catch(() => {});
     }
     ```
4. For ALL POST events (deauth or not), return `{ statusCode: 200, body: "EVENT_RECEIVED" }`. Strava sends many event types — only act on deauth, but always acknowledge.

**Other HTTP methods:**
Return `{ statusCode: 405, body: "Method Not Allowed" }`.

**Error handling:**
Wrap the deauth GitHub operations in a try/catch. On ANY error, console.error the message but still return 200 to Strava. Strava retries on non-200 responses, and retrying a failed delete is fine (idempotent).

**Important patterns:**
- Use v1 ESM syntax: `export const handler = async (event, _context) => { ... };`
- Add the standard file header comment matching other functions.
- Use `console.error` for GitHub API failures so they appear in Netlify function logs for debugging.
- The deauth flow (GET SHA + DELETE + build hook) should complete well within Strava's 2-second window — GitHub API calls take ~500ms each and there are only 2 sequential calls.
