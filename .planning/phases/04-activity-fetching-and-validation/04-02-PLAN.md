<task type="checkpoint:human-verify" gate="blocking">
  <what-built>
Complete activity fetch and validation function (`strava-fetch-activity.js`) that:
1. Parses a Strava activity URL to extract the numeric activity ID
2. Reads the session cookie and refreshes the OAuth token if needed
3. Fetches the full activity from Strava API with `include_all_efforts=true`
4. Validates the activity belongs to the authenticated athlete
5. Validates the activity date falls within June 6-7, 2026
6. Extracts sector efforts and KOM segment IDs
7. Returns structured error objects for all failure cases
  </what-built>
  <how-to-verify>
**Prerequisites:**
- `netlify dev` running via `volta run --node 22.22.2 npx netlify dev --no-open`
- STRAVA_REDIRECT_URI temporarily set to `http://localhost:8888/.netlify/functions/strava-callback` in `.env`
- Strava API app callback domain temporarily set to `localhost` in Strava dashboard

**Test 1: Full round-trip with a real activity**
1. Open `http://localhost:8888/.netlify/functions/strava-auth` in your browser to authenticate with Strava
2. After OAuth completes and you're redirected back, copy the `strava_session` cookie value from your browser's dev tools (Application > Cookies)
3. Use curl to POST a real activity URL (use one of YOUR recent Strava activities):
```
curl -s -X POST http://localhost:8888/.netlify/functions/strava-fetch-activity \
  -H "Content-Type: application/json" \
  -H "Cookie: strava_session=<YOUR_SESSION_COOKIE_VALUE>" \
  -d '{"activityUrl":"https://www.strava.com/activities/<YOUR_ACTIVITY_ID>"}' | jq .
```
4. Verify the response contains: `activityId`, `athleteId`, `movingTimeSeconds`, `startDateLocal`, `sectorEfforts` (object), `komSegmentIds` (array)
5. Verify `movingTimeSeconds` is a reasonable number (in seconds)

**Test 2: Wrong athlete check (if possible)**
- If you have a URL to another rider's public activity, POST it and verify `{ "error": "wrong_athlete" }` is returned

**Test 3: Date validation**
- The activity you tested likely falls OUTSIDE June 6-7, 2026 (since the event hasn't happened yet). Verify you get `{ "error": "wrong_date", "actualDate": "...", "expectedDates": ["2026-06-06","2026-06-07"] }` with the correct actual date shown.

**Test 4: Invalid URL**
```
curl -s -X POST http://localhost:8888/.netlify/functions/strava-fetch-activity \
  -H "Content-Type: application/json" \
  -H "Cookie: strava_session=<YOUR_SESSION_COOKIE_VALUE>" \
  -d '{"activityUrl":"not a strava url"}' | jq .
```
Verify: `{ "error": "invalid_url" }`

**After testing:** Remember to revert STRAVA_REDIRECT_URI in `.env` back to the production URL and revert the Strava dashboard callback domain.
