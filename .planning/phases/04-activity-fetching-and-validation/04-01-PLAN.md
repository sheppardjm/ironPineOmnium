  <action>
Create `netlify/functions/strava-fetch-activity.js` as a Netlify Functions v1 ESM handler (`export const handler`). This function accepts a POST request with `{ activityUrl: string }` in the body and returns JSON.

The function implements these sequential steps:

**1. Session read:**
- Parse `strava_session` cookie from `event.headers.cookie` using `cookie-es` `parse()`
- If no cookie or unparseable JSON, return `{ statusCode: 401, body: JSON.stringify({ error: "no_session" }) }`

**2. Token refresh:**
- Call `getValidAccessToken(session)` from `./lib/strava-tokens.js`
- If it throws, return `{ statusCode: 401, body: JSON.stringify({ error: "token_refresh_failed" }) }`
- Destructure `{ accessToken, refreshToken, expiresAt, updated }`
- If `updated === true`, prepare a `multiValueHeaders['Set-Cookie']` array with a re-serialized `strava_session` cookie (same flags as strava-callback.js: httpOnly, secure=!isLocal, sameSite="lax", path="/", maxAge=3600)

**3. URL parsing:**
- Read `activityUrl` from `JSON.parse(event.body || "{}")`
- Match against `/strava\.com\/activities\/(\d+)/`
- If no match, return `{ statusCode: 200, body: JSON.stringify({ error: "invalid_url" }) }`
- Extract `activityId = match[1]` (string)

**4. Strava API fetch:**
- `fetch(`https://www.strava.com/api/v3/activities/${activityId}?include_all_efforts=true`, { headers: { Authorization: `Bearer ${accessToken}` } })`
- Wrap in try/catch:
  - On network error (catch block): return `{ statusCode: 200, body: JSON.stringify({ error: "network_error" }) }`
  - On `!res.ok`:
    - 401 -> `{ error: "activity_not_found" }` (Strava uses 401 for both expired token and private/other-athlete activity; since we already refreshed the token, this means inaccessible)
    - 404 -> `{ error: "activity_not_found" }`
    - 429 -> `{ error: "rate_limited" }`
    - Other -> `{ error: "strava_api_error", statusCode: status }`
  - All validation errors use statusCode 200 (structured error pattern from research)

**5. Ownership check:**
- `String(activity.athlete.id) !== session.athleteId` -> return `{ error: "wrong_athlete" }`
- CRITICAL: activity.athlete.id is a number from the API, session.athleteId is a string. Must convert with String().

**6. Date validation:**
- Extract `localDateStr = activity.start_date_local.slice(0, 10)` (e.g., "2026-06-06")
- Do NOT use `new Date()` or timezone math. The `start_date_local` field's date portion IS the local date, despite the misleading Z suffix.
- `EVENT_DATES = ["2026-06-06", "2026-06-07"]`
- If `!EVENT_DATES.includes(localDateStr)`, return `{ error: "wrong_date", actualDate: localDateStr, expectedDates: EVENT_DATES }`

**7. Segment effort extraction:**
- Import `SECTOR_SEGMENT_IDS` and `KOM_SEGMENT_IDS` from `../../src/lib/segments.ts`
  - NOTE: Netlify Functions can import from src/ via relative path since the bundler resolves it at build time. Use the .ts extension in the import path.
- `efforts = activity.segment_efforts || []`
- Build `sectorEfforts` as `Record<string, number>`: iterate efforts, `String(effort.segment.id)` -> if in SECTOR_SEGMENT_IDS, add `{ [segId]: effort.elapsed_time }`
  - If a rider has multiple efforts on the same segment (e.g., rode it twice), keep only the FASTEST (lowest elapsed_time). This prevents double-counting and is fairest for scoring.
- Build `komSegmentIds` as `string[]`: iterate efforts, `String(effort.segment.id)` -> if in KOM_SEGMENT_IDS, push segId
  - Deduplicate (a rider may have multiple efforts on a KOM segment too)

**8. Return trimmed response:**
```javascript
{
  statusCode: 200,
  headers: { "Content-Type": "application/json" },
  multiValueHeaders: Object.keys(multiValueHeaders).length ? multiValueHeaders : undefined,
  body: JSON.stringify({
    activityId: String(activity.id),
    athleteId: session.athleteId,
    movingTimeSeconds: activity.moving_time,
    startDateLocal: localDateStr,
    sectorEfforts,
    komSegmentIds,
  }),
}
```
