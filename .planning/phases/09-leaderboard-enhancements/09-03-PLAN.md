<task type="checkpoint:human-verify" gate="blocking">
  <what-built>
Phase 9 leaderboard enhancements:
1. Per-component score columns with self-explanatory headers (Day 1, Day 2 Sectors, KOM, Total)
2. Client-side name search with real-time row filtering across all tabs
3. Mobile-optimized layout with sticky Rider column at 375px
4. Touch-safe interactions (no accidental triggers)
  </what-built>
  <how-to-verify>
Open the local preview URL in your browser and verify:

**SC-1 — Per-component columns:**
- Scroll to the leaderboard section
- Confirm headers read: # | Rider | Day 1 | Day 2 Sectors | KOM | Total
- Each column should show a numeric score value (no additional explanation needed)

**SC-2 — Name search:**
- Type a rider name in the search field above the tabs
- Rows should filter in real time as you type
- Switch to a different category tab — the filtered state should persist
- Clear the search field — all rows should reappear in all tabs

**SC-3 — Mobile layout at 375px:**
- Open Chrome DevTools (Cmd+Opt+I), toggle device toolbar (Cmd+Shift+M)
- Select iPhone SE (375px wide) or set custom width to 375
- Scroll to the leaderboard:
  - Rider name and Total score columns should be readable without horizontal scroll
  - If you scroll the table horizontally, the Rider column should stay fixed (sticky)
  - The sticky column background should be solid (no ghosted content behind it)
  - The `.score-note` sub-labels (raw duration/points) should be hidden on mobile

**SC-4 — Touch target safety:**
- Still at 375px viewport, tap on a leaderboard table row
- Nothing should happen (no navigation, no search input focus)
- Tap a category tab — it should respond immediately (no 300ms delay)
- Double-tap the search input — it should not zoom the page
  </how-to-verify>
