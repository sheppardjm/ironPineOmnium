**Part B — Search filtering script in Leaderboard.astro:**

Extend the existing `<script>` block. Inside the `for (const leaderboard of leaderboards)` loop, after the existing tab-switching logic, add:

```typescript
const searchInput = leaderboard.querySelector<HTMLInputElement>('[data-search]');
if (searchInput) {
  searchInput.addEventListener('input', () => {
    const query = searchInput.value.trim().toLowerCase();
    const allRows = leaderboard.querySelectorAll<HTMLTableRowElement>('tbody tr');
    for (const row of allRows) {
      const name = row.querySelector('.rider-name')?.textContent?.toLowerCase() ?? '';
      const matches = query === '' || name.includes(query);
      row.classList.toggle('search-hidden', !matches);
    }
  });
}
```

Key requirements:
- The query uses `querySelectorAll('tbody tr')` on the entire `leaderboard` element (not a single panel) — this filters rows across ALL panels simultaneously.
- Uses `classList.toggle('search-hidden', ...)` — NOT `row.hidden` which would conflict with the tab panel `hidden` attribute system.
- Empty query (`''`) shows all rows (removes `.search-hidden` from everything).
