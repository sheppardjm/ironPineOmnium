<task type="auto">
  <name>Task 1: Create strava-callback.js with full OAuth callback logic</name>
  <files>netlify/functions/strava-callback.js</files>
  <action>
Create `netlify/functions/strava-callback.js` using v1 ESM handler syntax (`export const handler`). Add the same style comment header as health.js and strava-auth.js explaining v1 ESM requirement.

The function receives Strava's redirect with query params `code`, `state`, `scope`, and potentially `error`. It must handle these steps in order:

**Step 1: Handle user denial**
- Check `event.queryStringParameters.error === 'access_denied'`
- If yes, redirect to `/error?reason=strava_denied`

**Step 2: Verify CSRF nonce**
- Import `parse` and `serialize` from `cookie-es`
- Parse cookies from `event.headers.cookie || ''` using `parse()`
- Extract `strava_csrf` cookie value
- Compare it to `event.queryStringParameters.state`
- If missing or mismatched, redirect to `/error?reason=csrf_mismatch`

**Step 3: Verify scope**
- Read `event.queryStringParameters.scope`
- Check that the scope string contains `activity:read_all` (Strava returns a comma-separated list of granted scopes like `read,activity:read_all`)
- If `activity:read_all` is NOT in the granted scopes, redirect to `/error?reason=insufficient_scope`

**Step 4: Exchange authorization code for tokens**
- POST to `https://www.strava.com/oauth/token` with JSON body:
  ```json
  {
    "client_id": process.env.STRAVA_CLIENT_ID,
    "client_secret": process.env.STRAVA_CLIENT_SECRET,
    "code": code,
    "grant_type": "authorization_code"
  }
  ```
- Use native `fetch` (Node 22 built-in)
- If response is not ok, redirect to `/error?reason=token_exchange_failed`
- Parse JSON response. Extract `athlete.id` (convert to string with `String()`), `access_token`, `refresh_token`, `expires_at`
- IMPORTANT: `athlete` object is ONLY in this initial exchange response, never in refresh responses

**Step 5: Set session cookie and clear CSRF cookie**
- Detect local dev: `const isLocal = process.env.NETLIFY_DEV === 'true'`
- Create session cookie payload as JSON string: `{ athleteId, accessToken, refreshToken, expiresAt }`
- Serialize session cookie: `serialize('strava_session', sessionPayload, { httpOnly: true, secure: !isLocal, sameSite: 'lax', path: '/', maxAge: 3600 })`
- Serialize clear CSRF cookie: `serialize('strava_csrf', '', { httpOnly: true, secure: !isLocal, sameSite: 'lax', path: '/', maxAge: 0 })`
- IMPORTANT: Use `multiValueHeaders` for the two Set-Cookie headers (NOT `headers['Set-Cookie']` which only keeps the last one):
  ```javascript
  return {
    statusCode: 302,
    headers: { Location: '/', 'Cache-Control': 'no-cache' },
    multiValueHeaders: { 'Set-Cookie': [sessionCookie, clearCsrf] },
    body: '',
  };
  ```
- Redirect to `/` for now (the submission page is built in Phase 5; this is a placeholder destination)

**Error handling:** Wrap the token exchange fetch in a try/catch. On network error, redirect to `/error?reason=token_exchange_failed`.

<task type="auto">
  <name>Task 2: Create strava-tokens.js token refresh utility</name>
  <files>netlify/functions/lib/strava-tokens.js</files>
  <action>
Create `netlify/functions/lib/strava-tokens.js` as a shared ESM module (not a Netlify Function — no `handler` export). This is a utility imported by downstream functions (Phase 4+) when they need to make Strava API calls.

The module exports one function:

```javascript
/**
 * Returns a valid access token, refreshing if needed.
 * @param {Object} session - { accessToken, refreshToken, expiresAt, athleteId }
 * @returns {Promise<{ accessToken, refreshToken, expiresAt, updated }>}
 *   `updated` is true if a refresh occurred (caller must update session cookie)
 */
export async function getValidAccessToken(session) { ... }
```

Implementation:
1. Extract `accessToken`, `refreshToken`, `expiresAt` from session
2. Calculate `nowUnix = Math.floor(Date.now() / 1000)`
3. Define `BUFFER_SECONDS = 300` (5-minute buffer)
4. If `expiresAt - nowUnix > BUFFER_SECONDS`, return `{ accessToken, refreshToken, expiresAt, updated: false }` (token still valid)
5. Otherwise, POST to `https://www.strava.com/oauth/token` with:
   ```json
   {
     "client_id": process.env.STRAVA_CLIENT_ID,
     "client_secret": process.env.STRAVA_CLIENT_SECRET,
     "grant_type": "refresh_token",
     "refresh_token": refreshToken
   }
   ```
6. If response is not ok, throw `new Error('Token refresh failed: ' + res.status)`
7. Parse JSON response. Extract `access_token`, `refresh_token` (CRITICAL: may be a NEW value — always update), `expires_at`
8. Return `{ accessToken: data.access_token, refreshToken: data.refresh_token, expiresAt: data.expires_at, updated: true }`
9. Add a JSDoc comment noting: "The athlete object is NOT included in refresh responses — only in the initial authorization_code exchange."

**Do NOT:**
- Export a `handler` — this is a library, not a function endpoint
- Import `cookie-es` — callers handle their own cookie reading/writing
- Store anything — this is a pure function; callers manage persistence

Add a file-level comment: "Shared token refresh utility for Netlify Functions. Not a function endpoint — no handler export."
