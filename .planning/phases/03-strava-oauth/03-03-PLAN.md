**Test 1: Happy path (approve consent)**
1. Open browser to `http://localhost:8888/api/strava-auth`
2. You should be redirected to Strava's OAuth consent screen
3. Verify the consent screen shows your app name and requests "View data about your activities"
4. Click "Authorize"
5. You should be redirected back to `http://localhost:8888/` (the homepage)
6. Open browser DevTools -> Application -> Cookies
7. Verify a `strava_session` cookie exists with HttpOnly flag
8. Verify the `strava_csrf` cookie is gone (cleared by callback)
9. Copy the `strava_session` cookie value — it should be a JSON string with `athleteId`, `accessToken`, `refreshToken`, `expiresAt`

**Test 2: Denial path**
1. First clear all cookies for localhost:8888
2. Open browser to `http://localhost:8888/api/strava-auth`
3. On the Strava consent screen, click "Cancel" or deny authorization
4. You should be redirected to `http://localhost:8888/error?reason=strava_denied`
5. Verify the error page shows a clear explanation and a "Try Again" link

**Test 3: Error page renders**
1. Visit `http://localhost:8888/error?reason=csrf_mismatch` directly
2. Verify it shows the CSRF-specific error message
3. Visit `http://localhost:8888/error?reason=insufficient_scope`
4. Verify it shows the scope-specific error message
