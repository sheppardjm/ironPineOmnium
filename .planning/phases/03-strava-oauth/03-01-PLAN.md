2. Create `netlify/functions/strava-auth.js` following the exact v1 ESM handler pattern from health.js (`export const handler`). The function must:

   - Import `crypto` from Node built-in and `serialize` from `cookie-es`
   - Generate a CSRF nonce: `crypto.randomBytes(32).toString('hex')`
   - Detect local dev: `const isLocal = process.env.NETLIFY_DEV === 'true'`
   - Create `strava_csrf` cookie with `serialize('strava_csrf', nonce, { httpOnly: true, secure: !isLocal, sameSite: 'lax', path: '/', maxAge: 600 })`
   - Build Strava authorization URL using `URLSearchParams` with:
     - `client_id`: `process.env.STRAVA_CLIENT_ID`
     - `redirect_uri`: `process.env.STRAVA_REDIRECT_URI`
     - `response_type`: `'code'`
     - `approval_prompt`: `'auto'`
     - `scope`: `'activity:read_all'`
     - `state`: the nonce
   - Return `{ statusCode: 302, headers: { Location: stravaAuthUrl, 'Set-Cookie': csrfCookie, 'Cache-Control': 'no-cache' }, body: '' }`

   Add a comment at the top matching health.js style explaining this is Netlify Functions v1 ESM and why `export default` is forbidden.

   Do NOT use `multiValueHeaders` — only one cookie is set here, so `headers['Set-Cookie']` is correct.
  </action>

<task type="auto">
  <name>Task 2: Create error.astro page for OAuth failure redirects</name>
  <files>src/pages/error.astro</files>
  <action>
Create `src/pages/error.astro` that handles all OAuth error scenarios via the `reason` query parameter.

The page must:
- Use the existing `BaseLayout` wrapper (import from `../layouts/BaseLayout.astro`)
- Read `Astro.url.searchParams.get('reason')` in the frontmatter
- Map reason codes to user-friendly messages in the frontmatter:
  - `strava_denied` -> "You declined the Strava authorization. We need access to your activity data to verify your ride and compute your score."
  - `csrf_mismatch` -> "Your authorization session expired or was invalid. This can happen if you waited too long or opened multiple tabs."
  - `token_exchange_failed` -> "We couldn't complete the connection to Strava. This is usually temporary."
  - `insufficient_scope` -> "We need the 'View your activity data' permission to read your ride details. Without it, we can't compute your score."
  - Default (unknown reason) -> "Something went wrong during the Strava connection."
- Render a centered card with:
  - A heading: "Connection Error"
  - The user-friendly error message
  - A "Try Again" link that goes to `/api/strava-auth` (re-initiates the OAuth flow)
  - A "Back to Home" link that goes to `/`
- Style with Tailwind classes matching the site's design language:
  - Use `font-display` (Cormorant Garamond) for the heading
  - Use `font-sans` (Sora) for body text
  - Dark background consistent with site theme (the site uses a dark pine/forest aesthetic)
  - Cards with subtle border and padding

Do NOT add any client-side JavaScript — this is a pure static page rendered at build time with Astro's built-in query parameter access.

Note: Astro static mode CAN read query params at build time for page generation, but for dynamic query params at runtime, Astro's `Astro.url` only works in SSR mode. Since this project is static output mode, the error page should use a client-side `<script>` tag to read `window.location.search` and update the displayed message. Keep the script minimal — just read the `reason` param and set the text content of the message element.
