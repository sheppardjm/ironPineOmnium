<task type="auto">
  <name>Task 1: Automated pre-flight checks — curl live site, verify HTML source, sitemap, and robots.txt</name>
  <files></files>
  <action>
Run the following curl commands against the live Netlify deployment at https://ironpineomnium.com and verify the output matches expectations. Do NOT test against localhost or dist/.

**Step 1 — Precondition: Confirm live site is current build**
```bash
curl -sI https://ironpineomnium.com | head -20
```
Confirm HTTP 200 response and Netlify headers present.

**Step 2 — Homepage head tag audit**
```bash
curl -s https://ironpineomnium.com | grep -iE 'og:|twitter:|canonical|application/ld\+json|"@type"'
```
Verify ALL of the following are present:
- `<meta property="og:image" content="https://ironpineomnium.com/og-image.png">`
- `<meta property="og:image:width" content="1200">`
- `<meta property="og:image:height" content="630">`
- `<meta name="twitter:card" content="summary_large_image">`
- `<script type="application/ld+json">` containing `"@type":"Event"` and `"@context":"https://schema.org"`
- `<link rel="canonical" href="https://ironpineomnium.com/">`

**Step 3 — Twitter card tags via Twitterbot user-agent**
```bash
curl -s -A "Twitterbot/1.0" https://ironpineomnium.com | grep -i "twitter:card\|twitter:image\|twitter:title"
```
Confirm `summary_large_image` card type and correct image URL.

**Step 4 — Sitemap index**
```bash
curl -s https://ironpineomnium.com/sitemap-index.xml
```
Confirm XML is served and references `https://ironpineomnium.com/sitemap-0.xml`.

**Step 5 — Sitemap page list**
```bash
curl -s https://ironpineomnium.com/sitemap-0.xml
```
Confirm exactly 4 URLs present:
- `https://ironpineomnium.com/`
- `https://ironpineomnium.com/leaderboard/`
- `https://ironpineomnium.com/submit/`
- `https://ironpineomnium.com/support/`
Confirm `/submit-confirm/` is ABSENT.

**Step 6 — Robots.txt**
```bash
curl -s https://ironpineomnium.com/robots.txt
```
Confirm contains `Sitemap: https://ironpineomnium.com/sitemap-index.xml` and `Allow: /`.

**Step 7 — submit-confirm noindex check**
```bash
curl -s https://ironpineomnium.com/submit-confirm/ | grep -i 'robots'
```
Confirm `<meta name="robots" content="noindex">` is present.

**Step 8 — All other pages have OG tags (spot check)**
For each of /leaderboard/, /submit/, /support/:
```bash
curl -s https://ironpineomnium.com/leaderboard/ | grep -c 'og:image'
curl -s https://ironpineomnium.com/submit/ | grep -c 'og:image'
curl -s https://ironpineomnium.com/support/ | grep -c 'og:image'
```
Each should return at least 1.

**Step 9 — OG image is accessible and correct size**
```bash
curl -sI https://ironpineomnium.com/og-image.png | grep -iE 'content-type|content-length|HTTP'
```
Confirm HTTP 200, content-type: image/png, file is served.

Report findings clearly: list each check as PASS or FAIL with the relevant output. If any check fails, report it but continue all remaining checks.

  <name>Task 3: Facebook Sharing Debugger — verify OG tags and force rescrape</name>
  <what-built>All OG tags (og:image, og:title, og:description, og:url, og:type) are implemented on all pages. Task 1 confirmed the live HTML serves these tags correctly.</what-built>
  <how-to-verify>
**Requirement: QA-01**

**Option A — Facebook Sharing Debugger (authoritative, requires Facebook login):**
1. Log in to Facebook at facebook.com
2. Navigate to https://developers.facebook.com/tools/debug/
3. Paste `https://ironpineomnium.com` into the URL field
4. Click **Debug**
5. Review the "Link Preview" section — verify correct image, title, description
6. Click **Scrape Again** (CRITICAL — this forces Facebook to re-read the live page and clears its cache)
7. Re-verify the preview refreshes correctly after rescrape

**Option B — opengraph.to (no login required, equivalent preview but does NOT force Facebook cache clear):**
1. Navigate to https://www.opengraph.to/
2. Paste `https://ironpineomnium.com`
3. Verify the Facebook preview tab shows: correct OG image (1200x630), correct title, correct description

**What PASS looks like:**
- OG image renders at correct proportions (wide rectangle, not cropped/stretched)
- Title shows "Iron and Pine Omnium | Two Days of Gravel in the Hiawatha National Forest" (or similar — the exact og:title from the homepage)
- Description is the event summary, not blank or a URL
- No "Missing Required Property" warnings for og:image, og:title, or og:description

**What FAIL looks like:**
- Missing or broken image preview
- Wrong title or description
- "Could not fetch" error
- "og:image could not be downloaded" warning

**Note:** If using Option B only, the Facebook cache has not been cleared. Before sharing the URL on Facebook for the first time, you should run Option A to force the rescrape.
  </how-to-verify>
  <resume-signal>Type "pass" (and note which option used), or describe the failure</resume-signal>
