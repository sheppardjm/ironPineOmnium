**Create src/components/PodiumPreview.astro:**

Build-time component that loads athlete results and shows top 3 per category in a podium-style layout. Uses `loadAthleteResults()` and `scoreOmnium()` from the existing scoring library — same data source as Leaderboard.astro.

Structure:
```astro
---
import { defaultScoringConfig, scoreOmnium } from "../lib/scoring";
import { loadAthleteResults } from "../lib/athlete-loader";

const { riders, hasLiveData } = loadAthleteResults();
const leaderboards = scoreOmnium(riders, defaultScoringConfig);

const scoreFormatter = new Intl.NumberFormat("en-US", {
  minimumFractionDigits: 1,
  maximumFractionDigits: 1,
});

const podiumData = leaderboards.map(board => ({
  categoryLabel: board.categoryLabel,
  categoryId: board.categoryId,
  top3: board.entries.slice(0, 3),
}));
---
```

Layout:
- If `hasLiveData` is false, show a placeholder: "Leaderboard opens after submissions begin" with a muted label, and a link to /leaderboard. Style this placeholder with `--color-ink-muted` text on the section background.
- If `hasLiveData` is true, render a grid of podium cards — one per category (3 columns on desktop, stacked on mobile). Each card shows:
  - Category name as header (e.g., "Men", "Women", "Non-Binary")
  - Top 3 riders: rank (1, 2, 3), name, total score (formatted via scoreFormatter)
  - Rank 1 can be slightly larger/emphasized
- Below the podium grid, a "View Full Leaderboard" link (`<a href="/leaderboard" class="primary-button">`)

Style the podium using inline `<style>` in the component (scoped). Use the light-mode color tokens from global.css. The podium cards should have:
- Background: `var(--color-surface-raised, #ffffff)`
- Border: subtle `rgba(19, 28, 26, 0.08)`
- Rank numbers: bold Spectral font
- Rider names: Karla, medium weight
- Score: JetBrains Mono, tabular

Keep the component simple. No tabs. No search. Just a clean top-3 snapshot.
