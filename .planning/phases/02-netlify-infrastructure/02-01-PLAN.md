  <files>netlify.toml, netlify/functions/health.js</files>
  <action>
Create `netlify.toml` at project root with the following sections:

```toml
[build]
  command = "pnpm run build"
  publish = "dist"

[functions]
  directory = "netlify/functions"
  node_bundler = "esbuild"

[[redirects]]
  from = "/api/*"
  to = "/.netlify/functions/:splat"
  status = 200
  force = true

[dev]
  command = "pnpm run dev"
  targetPort = 4321
  port = 8888
  publish = "dist"
  framework = "#custom"
```

Create directory `netlify/functions/` and write `health.js`:

```javascript
// Netlify Functions v1 (Lambda-compatible, ESM)
// Uses `export const handler` — NOT `export default` (which is v2, forbidden).
// This project has "type": "module" in package.json, so .js files are ESM.
export const handler = async (event, context) => {
  return {
    statusCode: 200,
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ ok: true }),
  };
};
```

IMPORTANT: Do NOT use `export default` (that is v2 syntax and forbidden by project decision). Do NOT use `exports.handler` in a `.js` file (will fail because package.json has `"type": "module"`). The correct v1 ESM syntax is `export const handler`.
  </action>

- `cat netlify/functions/health.js` contains `export const handler` and NOT `export default`
- The redirect rule maps `/api/*` to `/.netlify/functions/:splat` with status 200
  </verify>
  <done>netlify.toml exists with correct build, functions, redirects, and dev config. health.js exists with v1 ESM handler syntax.</done>
