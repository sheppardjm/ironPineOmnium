Create `src/lib/athlete-loader.ts` with the following:

**AthleteJson interface** — matches the schema written by `submit-result.js`:
```typescript
export interface AthleteJson {
  athleteId: string;
  displayName: string;
  category: string; // "men" | "women" | "non-binary"
  day1: {
    movingTimeSeconds: number;
    activityId: string;
    submittedAt: string;
  } | null;
  day2: {
    sectorEfforts: Record<string, number>;
    komSegmentIds: string[];
    komEfforts?: Record<string, number>; // optional — present in Strava submissions, absent in CSV fallback
    activityId: string;
    submittedAt: string;
  } | null;
  createdAt: string;
  updatedAt: string;
}
```

**computeKomPoints(athlete, peersInCategory)** — internal function:
- If `athlete.day2` is null, return 0.
- If `athlete.day2.komEfforts` exists and has at least one key:
  - Use time-based ranking: for each KOM segment in `KOM_SEGMENT_IDS`, find the athlete's elapsed time. Count how many peers in the same category have a strictly faster time on that segment. Points for this segment = `ridersWithTime.length - fasterCount` (rank 1 gets max points, last gets 1). Sum across all KOM segments where the athlete has a time.
  - If the athlete has no time for a given KOM segment, skip that segment (0 points for it).
- If `komEfforts` is absent or empty: fall back to `athlete.day2.komSegmentIds?.length ?? 0`. This gives CSV fallback riders credit for KOM presence.

**loadAthleteResults()** — exported function:
1. Call `import.meta.glob('../../public/data/results/athletes/*.json', { eager: true })` to load all athlete JSON files. The glob path is relative to `src/lib/` — two levels up reaches `public/`.
2. Extract raw objects: `Object.values(athleteFiles).map((mod) => (mod as { default: AthleteJson }).default)`.
3. Filter to complete athletes: keep only those where `day1 !== null && day2 !== null`.
4. Validate category: keep only those where `category` is one of `categoryIds` from `types.ts`.
5. For each complete athlete, compute KOM points by passing the athlete and all complete same-category athletes to `computeKomPoints`.
6. Map to `RiderResult[]`:
   - `id` = `athleteId`
   - `name` = `displayName`
   - `category` = cast to `CategoryId`
   - `movingTimeSeconds` = `day1.movingTimeSeconds`
   - `sectorTimesSeconds` = `SECTOR_SEGMENT_IDS.map(segId => day2.sectorEfforts[segId] ?? 0)`
   - `komPoints` = result of `computeKomPoints`
7. Return the `RiderResult[]`.

Also return `hasLiveData: boolean` alongside the results. Export as:
```typescript
export function loadAthleteResults(): { riders: RiderResult[]; hasLiveData: boolean }
```
Where `hasLiveData = riders.length > 0`.

**Imports needed:**
- `{ KOM_SEGMENT_IDS, SECTOR_SEGMENT_IDS }` from `./segments`
- `type { RiderResult, CategoryId }` and `{ categoryIds }` from `./types`

Do NOT modify `scoring.ts`. Do NOT import `defaultScoringConfig` — this layer produces raw data for the scoring engine, it does not apply weights.
