# Codebase Structure

**Analysis Date:** 2026-04-02

## Directory Layout

```
ironPineOmnium/
├── src/                       # Source code
│   ├── components/            # Reusable Astro components
│   ├── layouts/               # Page layout templates
│   ├── lib/                   # Business logic and utilities
│   ├── pages/                 # File-based routing (becomes HTTP routes)
│   ├── styles/                # Global CSS and design tokens
│   ├── env.d.ts               # Astro type definitions
├── public/                    # Static assets served as-is
│   ├── images/                # Route photography assets
│   └── logo.svg               # Brand mark
├── dist/                      # Generated static HTML (build output)
├── .astro/                    # Generated Astro metadata
├── astro.config.mjs           # Astro configuration
├── tsconfig.json              # TypeScript configuration
├── package.json               # Dependencies and scripts
└── pnpm-lock.yaml             # Locked dependency versions
```

## Directory Purposes

**`src/`:**
- Purpose: All source code and styling
- Contains: Components, layouts, pages, utilities, styles
- Key files: All application code lives here

**`src/components/`:**
- Purpose: Reusable Astro components for building pages
- Contains: `.astro` files (server-rendered, static components)
- Key files: `Leaderboard.astro` (main data table), `LogoMark.astro` (SVG icon component)

**`src/layouts/`:**
- Purpose: Page layout wrapper components that define page structure
- Contains: Layout Astro components used by pages
- Key files: `BaseLayout.astro` (wraps all pages with HTML structure, meta tags, global styles)

**`src/lib/`:**
- Purpose: Business logic, types, and data utilities
- Contains: TypeScript modules (scoring engine, type definitions, sample data)
- Key files: `scoring.ts` (omnium calculation), `types.ts` (data interfaces), `sample-data.ts` (12 rider test data)

**`src/pages/`:**
- Purpose: File-based routing - each file becomes an HTTP route
- Contains: Astro page components
- Key files: `index.astro` (root `/` page - the entire omnium landing site)

**`src/styles/`:**
- Purpose: Global styling and design system
- Contains: CSS with Tailwind directives and custom properties
- Key files: `global.css` (theme tokens, animations, Tailwind imports)

**`public/`:**
- Purpose: Static assets served directly, not processed by build
- Contains: Images, logos, files that don't need transformation
- Key files: `images/` (4 route photography JPGs), `logo.svg` (brand mark)

**`dist/`:**
- Purpose: Build output directory (generated by Astro build)
- Contains: Static HTML, CSS, and optimized assets
- Key files: `index.html` (rendered page)

**`.astro/`:**
- Purpose: Generated directory with Astro metadata and type definitions
- Contains: Auto-generated type files, font data, settings
- Key files: `types.d.ts` (component type definitions), `content.d.ts` (content collection types)

## Key File Locations

**Entry Points:**
- `src/pages/index.astro`: Root page `/` - entire omnium landing site with hero, leaderboard, event details
- `astro.config.mjs`: Build configuration, Vite plugins, font providers

**Configuration:**
- `tsconfig.json`: TypeScript compiler options (strict mode, ESNext target)
- `package.json`: Dependencies (Astro, Tailwind), scripts, Node version requirement
- `astro.config.mjs`: Astro build config, Tailwind CSS integration, Google Fonts

**Core Logic:**
- `src/lib/scoring.ts`: Scoring engine - calculates omnium results per category
- `src/lib/types.ts`: TypeScript interfaces - RiderResult, ScoringConfig, CategoryLeaderboard
- `src/lib/sample-data.ts`: 12 test riders across 3 categories (men, women, non-binary)

**Components:**
- `src/components/Leaderboard.astro`: Renders category leaderboards with tabbed switcher and data table
- `src/components/LogoMark.astro`: SVG component for brand mark icon
- `src/layouts/BaseLayout.astro`: Root HTML layout, meta tags, global styles

**Styling:**
- `src/styles/global.css`: Tailwind import, CSS custom properties, keyframe animations
- Tailwind config: Embedded in `astro.config.mjs` via Vite plugin

**Testing:**
- No test files present (static site, no test suite configured)

## Naming Conventions

**Files:**
- Pages: `index.astro` (root), kebab-case for nested routes
- Components: PascalCase (e.g., `Leaderboard.astro`, `LogoMark.astro`)
- Utilities: camelCase (e.g., `scoring.ts`, `types.ts`, `sample-data.ts`)
- Styles: `global.css` for site-wide styles
- Layouts: PascalCase (e.g., `BaseLayout.astro`)

**Directories:**
- Pluralized: `src/components/`, `src/pages/`, `src/layouts/`, `src/styles/`, `public/images/`
- Special: `.astro/` for generated metadata, `dist/` for build output

**TypeScript Naming:**
- Interfaces: PascalCase (e.g., `RiderResult`, `ScoringConfig`, `CategoryLeaderboard`)
- Constants: camelCase (e.g., `defaultScoringConfig`, `categoryIds`)
- Functions: camelCase (e.g., `scoreOmnium()`, `scoreCategory()`, `formatDuration()`)
- Types (unions/aliases): PascalCase (e.g., `CategoryId`)

## Where to Add New Code

**New Feature (omnium-related):**
- Primary code: `src/lib/[feature-name].ts` (if logic), or extend existing `src/lib/scoring.ts`
- Components: `src/components/[FeatureName].astro`
- Page sections: Add to `src/pages/index.astro` template

**New Component:**
- File: `src/components/[ComponentName].astro`
- Pattern: Copy structure from `src/components/Leaderboard.astro` (frontmatter + template)
- Register: Import and use in `src/pages/index.astro` or other pages/layouts

**New Page:**
- File: `src/pages/[route-name].astro`
- Pattern: Import layout, add content, export HTML
- Layout: Use `BaseLayout` as wrapper via `<BaseLayout><main>...</main></BaseLayout>`

**Utilities:**
- Shared helpers: `src/lib/[name].ts`
- Scoring logic: Extend `src/lib/scoring.ts` with new functions
- Types: Add interfaces to `src/lib/types.ts`

**Styling:**
- Global styles: Add to `src/styles/global.css`
- Component styles: Inline `class:list` with Tailwind classes in `.astro` files
- Design tokens: Add CSS custom properties to `@theme` block in `src/styles/global.css`

**Static Assets:**
- Images: Place in `public/images/`
- Documents/files: Place in `public/`
- Access: Reference as `/images/filename.jpg` in HTML (Astro resolves public path)

## Special Directories

**`.astro/`:**
- Purpose: Generated directory with type definitions and settings
- Generated: Yes - auto-created by Astro during build
- Committed: No - included in `.gitignore`

**`dist/`:**
- Purpose: Static site output (production-ready HTML + CSS)
- Generated: Yes - created by `npm run build`
- Committed: No - build artifact

**`node_modules/`:**
- Purpose: Installed dependencies
- Generated: Yes - created by `pnpm install`
- Committed: No - restored from lock file
